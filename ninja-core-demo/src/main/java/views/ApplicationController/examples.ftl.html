Session support

  • The default mode of Ninja is to store the session as signed cookie on the client side. Check out /session

Flash cookie support

  • To transmit small bits of information from one request to another Ninja supports so called flash cookies. They transmit eg a error message from one request to another. Usually flash cookies only live for one request.
  • Success message via flash cookie: /flash_success
  • Error message via flash cookie: /flash_error
  • An arbitrary flash message: /flash_any
